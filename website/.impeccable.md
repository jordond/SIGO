## Design Context

### Users

Casual, fun-seeking people who want a quick, playful answer to "should I go outside?" They're
checking in briefly — the experience should feel lighthearted and delightful, not utilitarian.

### Brand Personality

**Fun, bold, friendly.** Energetic and warm with an approachable voice. The app doesn't take itself
too seriously but always delivers a clear answer.

### Aesthetic Direction

**Neobrutalism** — chunky 3px black borders, hard drop shadows (4px/8px offsets), bright multi-color
palette on a warm cream background. Tilted elements, playful geometry, and bouncy animations create
energy without chaos.

- **Palette**: Yellow `#ffdb02` (primary), sky blue `#87ceeb`, hot pink `#ff69b4`, purple `#a388ee`,
  coral `#ff6b6b`, mint `#90ee90`, orange `#ff8c42` on cream `#feecde`, surface `#fff5ea`
- **Typography**: Lexend Mega 700 for display/headings (tight letter-spacing via `--display-tracking: -0.16em`), Public Sans (400–700) for body/content — matching the native app
- **Interactions**: Lift-on-hover, press-in-on-click, slide-in underlines, cursor-reactive parallax shapes, staggered entrance animations
- **Theme**: Light mode only. The warm cream background is core to the identity.
- **No CSS frameworks** — pure scoped Astro CSS with global CSS variables in `Layout.astro`

### Design Principles

1. **Personality over polish** — Every element should feel handcrafted and slightly imperfect.
   Tilted cards, offset shadows, and bold colors over sleek gradients.
2. **One glance, one answer** — The app exists to give a yes/no answer fast. Visual hierarchy must
   make the core content unmissable.
3. **Color with purpose** — The bright palette is a feature, not decoration. Each color signals
   meaning (primary actions, categories, states). Don't muddy it.
4. **Playful but accessible** — Animations and personality never compromise usability. Respect
   `prefers-reduced-motion`, maintain WCAG AA contrast, keep interactions predictable.
5. **Less is more** — Resist adding complexity. If a section doesn't help someone decide to download
   the app, it probably doesn't belong.

### Design System Tokens

All tokens are defined as CSS custom properties in `Layout.astro` `:root`.

#### Spacing Scale

| Token | Value | Role |
|-------|-------|------|
| `--space-xs` | 0.5rem | Tight siblings (list items, icon gaps) |
| `--space-sm` | 1rem | Related content, button groups |
| `--space-md` | 1.5rem | Card padding, section inline padding |
| `--space-lg` | 2rem | Content groups, grid gaps |
| `--space-xl` | 3rem | Section content gaps, major separations |
| `--space-2xl` | 5rem | Hero/CTA vertical padding |
| `--space-3xl` | 7rem | Large section margins |
| `--space-section` | clamp(4rem, 8vw, 7rem) | Fluid section padding |
| `--space-section-lg` | clamp(5rem, 10vw, 9rem) | Fluid large section padding |

#### Type Scale

| Token | Value | Role |
|-------|-------|------|
| `--text-xs` | 0.75rem | Copyright, fine print, eyebrow pills |
| `--text-sm` | 0.875rem | Card descriptions, nav links, labels |
| `--text-base` | 1rem | Body text, buttons |
| `--text-md` | 1.125rem | Card titles, CTA subtitle, platform names |
| `--text-lg` | 1.25rem | Taglines, lead text |
| `--display-tracking` | -0.16em | Lexend Mega letter-spacing (all sizes) |
| `--display-leading` | 0.92 | Lexend Mega line-height (all sizes) |

Display headings use `clamp()` for fluid sizing. Body text uses fixed `rem` values.

#### Weight Roles

| Weight | Role |
|--------|------|
| 400 | Body text, descriptions |
| 600 | Labels, emphasized text, nav links |
| 700 | Headings, buttons, card titles |

#### Brutal Tokens

| Token | Value |
|-------|-------|
| `--border-width` | 3px |
| `--shadow-offset` | 4px |
| `--shadow-offset-lg` | 8px |
| `--radius-sm` | 4px |
| `--radius-md` | 8px |
| `--radius-lg` | 16px |

#### Z-Index Scale

| Token | Value | Role |
|-------|-------|------|
| `--z-base` | 1 | Content elements |
| `--z-sticky` | 100 | Sticky header |
| `--z-overlay` | 200 | Modals, overlays |

### Section Rhythm

The homepage alternates backgrounds to create visual rhythm:
- **Hero**: cream (`--color-bg`) — animated shapes provide visual energy
- **FeatureShowcase**: surface (`--color-surface`) with top/bottom borders
- **TrackedConditions**: cream (`--color-bg`)
- **CTA**: yellow (`--color-primary`) — strongest visual break
- **Footer**: dark (`--color-text`)
